# Plantilla para crear prompts en 5 pasos

Esta plantilla se basa en 5 bloques: **Rol**, **Contexto**, **Tareas**, **Formato** y **Restricciones**.

---

1. Rol del modelo: Describe quién debe ser el modelo (perfil, seniority, especialización y objetivo principal).
2. Contexto Da contexto suficiente para que el modelo no tenga que adivinar: entorno, audiencia, información relevante y qué le vas a dar como entrada.
3. Tareas específicas Define exactamente qué debe hacer el modelo, en pasos numerados.
4. Formato de salida Especifica con precisión cómo quieres que sea la respuesta: tipo y estructura.
5. Restricciones y estilo Define límites, estilo, extensión y cosas que no debe hacer el modelo.
---
## Plantilla
```text
### Rol del modelo
Eres un/a [rol o perfil] con experiencia en [área/tecnología/sector] y especializado/a en [detalle específico].
Tu objetivo principal es [objetivo principal del rol].

### Contexto
Contexto del problema o necesidad:
- Sistema / entorno: [describir sistema, plataforma, entorno]
- Audiencia objetivo: [perfil de quién leerá la salida]
- Información relevante adicional: [puntos clave, restricciones de negocio, etc.]
- Entradas que proporcionaré: [tipo de datos, documentos, logs, requisitos, etc.]

### Tareas
Realiza las siguientes tareas de forma ordenada:
1. [Tarea 1: qué debe hacer primero]
2. [Tarea 2: análisis / transformación / propuesta]
3. [Tarea 3: validación / recomendaciones / resumen]
4. [Tarea 4 opcional: mejoras, riesgos, siguientes pasos]

### Formato de salida
La respuesta debe cumplir este formato:
- Tipo de salida: [texto libre / lista / tabla Markdown / JSON / YAML]
- Estructura:
  - Sección 1: [nombre] → [qué debe contener]
  - Sección 2: [nombre] → [qué debe contener]
  - Sección 3: [nombre] → [qué debe contener]
  
### Restricciones y estilo
Ten en cuenta las siguientes restricciones:
- Idioma: [idioma deseado].
- Nivel técnico: [básico / intermedio / avanzado] adaptado a [audiencia].
- Extensión: [máx. X palabras / X apartados / X viñetas].
- Estilo: [formal / cercano / directo / técnico / divulgativo].
- Prohibido: [qué no debe hacer].
- Otros límites: [no incluir credenciales, no ejecutar acciones destructivas, etc.].
```
